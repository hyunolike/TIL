## ✔HTTP통신
### 통합 프로토콜 HTTP란
- HTTP는 클라이언트 서버 프로그램
- `클라이언트`가 서버에 요청하는 방식
- 착각하지마
  > 내가 한번 요청할께 너가 두번줘 /너(클라이언트)가 나(서버)한테 데이터를 줘
- TCP기반

### HTTP Message
- Status Line or Start Line(상태줄, 시작줄)
- Header(헤더)
  > NAME : HYUNHO
- Body(본문) `유동적이다`
  > 서버/클라이언트가 주고받는 `실제 데이터`가 있는 부분

## ✔HTTP 메소드
- GET `주로 다룰꺼🤗`
- POST `주로 다룰꺼🤗`
- PUT `전체 리소스 갱신`
- PATCH `일부 리소스 수정`
- DELETE `리소스 삭제`
- HEAD
- OPTIONS
- TRACE
- CONNECT

### GET
파라미터의 정보가 URL에 표시가 된다.
  > 이것을 `쿼리스트링` 이라고 부룬다.
- __쿼리스트링__  `?`기준으로 쿼리스트링이 시작된다.
```
http://hostname[:post]/folder/folder/file?val1=a&val2=b ...
```

`대용량데이터` 저장 불가능

### POST
`리소스` 생성 변경하기위해 생긴거야 <br>
기존 파라미터 값을 body에 저장!!!
따라서 `대용량데이터` 저장가능

이외 메소드는 구글에서 찾아봐요~~

## ✔SSR vs CSR
- SSR (Server Side Rendering)
- CSR (Client Side Rendering)

`프레임워크` 배우기전 중요한 개념이라 할 수 있다.

### SSR
말 그대로 `서버`에서 랜더링 하는거야

### CSR
최근에 `모바일 시장`이 커지면서 새로운 기법 도입
> 여기서 __SPA__ 를 알야야 되!!
> 단위 페이지 어플리케이션이라고 해

요즘 `웹개발 트랜드` __SPA__
- 이전 방식은 계속 `새로고침`😱
-  요즘은 AJAX 이용해 새로 바인딩 제공(데이터 갈아끼우기)

`프레임워크 변화`

| 초기 | 후기     |
| :------------- | :------------- |
| Backbone.js, Augular.js       | React.js, Vue.js, Angular2       |

맨처음 프레임워크 나왔을 떄 점점 `Client Side`가 무거워지기 시작<br>
그래서 `View`만 따로 관리하자!!! 그래서 __후기 프레임워크__ 가 등장하기 시작

## ✔플랫폼(Flatform)
포스팅, 퍼블리싱 할 때 사용하는 거야<br>
플랫폼 사용없이 서버를 직접 구축해도 되~(개인서버 원하는 경우)<br>
`하지만` 그렇게 해야 할 이유도 돈도 없다!!!
하드웨어 가격만 50만원...구축하는데 일주일 넘게 걸려(데이터베이스 연결,공유기 바인딩, 방화벽 설치 등)
> 이럴때 사용하는 것이 `플랫폼`

### 클라우드 플랫폼 소개🎉
#### AWS(아마존 웹 서비스)
- 생각보다 자주 사용하면서 만만하다~(다른 플랫폼에 비해서ㅋㅋㅋ)
- 정말 원하는 모든 기능을 제공~~~
- 처음 사용했을 때 진입벽이 높아ㅜㅜ(코딩하는것도 아닌데...)쓰기 힘들게 만들었다.
- 대학생이 아니면 쓰기가 힘들다. 성인이면 돈이 나가는 플랫폼(비싼 플랫폼)
- 자격증도 있다.

#### Heroku
- 최초의 클라우드 플랫폼
- 쓰기가 쉽다
- 간단한 퍼블리싱 한다면 추천
- 트래픽이 넘치면 돈이 나가는 시스템
- 파워플한 기능이 있는건 아니다.

#### FireBase
- 정말 많은 사람들이 사용했을꺼야
- 모바일앱 플랫폼
- 소규모 스타트업에서 애용
- 가장 큰 특징이 서버리스 구조
- 개발자가 느끼기에 서버가 없다??
- 돈이 안나온다.

이것말고도 네이버, 구글, KT(기업용) 등 많이 있다.
우리가 쓰기좋은거 위에 3가지!!!
